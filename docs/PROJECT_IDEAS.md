# Идеи по всему проекту (RC Scheduler)

Общие идеи для улучшения приложения: для всех пользователей, UI/UX и по ролям (SUP, ADM, VOL). Только идеи, реализация — отдельно.

---

## Для всех пользователей

### Вход и регистрация
- **«Забыли пароль»** — ссылка на странице входа с восстановлением через email (если есть почтовый сервер или внешний сервис).
- **Запоминание email** — сохранять последний введённый email в `localStorage`, подставлять при следующем визите.
- **Регистрация по инвайту** — опционально: регистрация только по ссылке-приглашению (для закрытых интенсивов).
- **Валидация пароля при регистрации** — показывать требования (длина, символы) и индикатор силы в реальном времени (частично есть в настройках).

### Дашборд (главная)
- **Быстрые действия** — кнопки «Добавить пространство», «Календарь», «Последние сообщения» без лишних переходов.
- **Виджеты настраиваемые** — возможность свернуть/развернуть блоки (статистика, последние сообщения, пространства).
- **Фильтр «Мои / Все»** — для SUP уже есть фильтр по пользователю для сообщений; для остальных — явный переключатель «только мои сообщения» при наличии общего канала.
- **Ссылки из карточек** — клик по «последнее сообщение» или по пространству ведёт на страницу пространства или календарь с фильтром.
- **Обновить** — кнопка «Обновить» на дашборде без перезагрузки страницы.

### Пространства (список)
- Уже много сделано: поиск, сортировка, избранное, группы, режимы отображения, роль-специфика.
- **Группы по названию** — группировка по первому слову или тегу (например, «Февральский», «Мартовский»).
- **Быстрый переход в календарь** — с карточки передавать `workspaceId` в календарь (уже есть ссылка «Календарь»).

### Пространство (страница [id])
- См. `WORKSPACE_DETAIL_PAGE_IDEAS.md` — хлебные крошки, календарь, статистика, каналы, сообщения, экспорт для SUP и т.д. Часть уже реализована.

### Календарь
- Уже есть: роли/цвета, легенда, фильтры, неделя/месяц, обновить, сброс фильтров, «Запланировать», печать.
- **Напоминания** — опционально: за N минут до отправки показывать уведомление (браузерные или в интерфейсе).
- **Дублирование события** — «Создать копию» сообщения на другую дату/канал.
- **Фильтр по тексту** — поиск по тексту сообщения в календаре.

### Сообщения (страница /dashboard/messages)
- Сейчас страница минимальна; при наличии — **единый список сообщений** по всем пространствам с фильтром по пространству, каналу, статусу, дате.
- **Быстрый переход** — клик по сообщению открывает страницу пространства и при необходимости вкладку «Сообщения» или диалог редактирования.

### История (активность)
- **Фильтр по типу события** — вход, создание пространства, сообщение отправлено и т.д.
- **Фильтр по пользователю** — для SUP уже может быть; для себя — только свои события.
- **Экспорт** — кнопка «Скачать лог» (CSV/JSON) за выбранный период.

### Настройки
- **Смена пароля** — уже есть; можно добавить проверку «текущий пароль» перед сменой.
- **Тема** — светлая/тёмная/системная (если ещё не везде).
- **Язык** — переключатель RU/EN при появлении локализации.
- **Уведомления** — включить/выключить тосты или браузерные уведомления о запланированных отправках (если добавится).
- **Двухфакторная аутентификация** — опционально для приложения (не только для RC).

### Заблокированная страница (/dashboard/blocked)
- **Контакт администратора** — ссылка «Написать администратору» (mailto или инструкция).
- **Понятный текст** — различать «заблокирован» и «срок доступа истёк» (уже есть).

### Общее по приложению
- **Онлайн/офлайн** — индикатор в шапке или сайдбаре («Нет соединения»), повтор запросов при восстановлении сети (частично есть OfflineDetector).
- **Единый обработчик ошибок API** — при 401 редирект на логин, при 403 — сообщение «Нет доступа», при 5xx — «Ошибка сервера, попробуйте позже».
- **Клавиатурные сокращения** — например, Ctrl+K поиск, Escape закрытие модалок (по желанию).

## UI/UX

### Внешний вид
- **Единый стиль карточек** — `border-muted`, лёгкая тень при hover на всех страницах (дашборд, пространства, календарь, админка).
- **Цветовая система** — один акцентный цвет для «действия» (например, кнопка «Запланировать»), нейтральные для вторичных кнопок.
- **Скелетоны** — везде, где есть асинхронная загрузка (списки, таблицы, дашборд), единый вид скелетона.
- **Типографика** — единые размеры заголовков (h1/h2/h3), отступы между секциями (space-y-6 или 8).
- **Иконки** — везде Lucide, один размер в одном контексте (например, 4 в кнопках, 5 в карточках).

### Навигация
- **Хлебные крошки** — на всех вложенных страницах: Дашборд → Пространства → Название; Дашборд → Админка → Пользователи → Имя.
- **Ссылка «Назад»** — где уместно, оставлять явную кнопку или хлебные крошки.
- **Сайдбар** — уже есть свёртывание; можно подсветку текущего раздела и бейдж «новое» или счётчики (ожидающие сообщения и т.д.).

### Интерактив
- **Тултипы** — на всех иконках и кнопках без текста (Обновить, Копировать, Избранное, Проверить подключение).
- **Подтверждение опасных действий** — удаление, блокировка, смена роли, архивация — всегда AlertDialog с текстом последствия.
- **Состояния кнопок** — loading (спиннер), disabled с причиной (например, «Нет каналов»).
- **Пустые состояния** — везде иконка + короткий текст + призыв к действию (например, «Добавить пространство»).

### Адаптив
- **Мобильный сайдбар** — drawer/sheet на маленьких экранах вместо постоянного сайдбара.
- **Таблицы** — на узких экранах переключаться в карточки или горизонтальный скролл с фиксированной первой колонкой.
- **Календарь** — на телефоне по умолчанию неделя или день, месяц — свайпом.
- **Формы** — поля в колонку на мобильных, кнопки на всю ширину.

### Доступность и производительность
- **aria-labels** — для кнопок-иконок и навигации (для скринридеров).
- **Виртуализация** — длинные списки (каналы, сообщения, пользователи в админке) при росте данных.
- **Ленивая загрузка** — тяжёлые вкладки (импорт эмодзи, добавление пользователей) подгружать при первом открытии.

## По ролям

### SUP (Super Admin)
- **Админка** — уже есть пользователи, аудит, блокировка, роли, продление VOL; идеи из прошлых обсуждений: массовые действия, экспорт пользователей, фильтры.
- **Пространства** — быстрая архивация, экспорт каналов/сообщений в CSV, «Повторить отправку» для сообщений с ошибкой (если появится API).
- **Общая видимость** — видеть последние действия любого SUP по пространству (уже сделано).
- **Дашборд** — фильтр по пользователю для сообщений, сводка по всем пользователям (количество активных, VOL с истекающим доступом).
- **Журнал аудита** — фильтр по типу события, по пользователю, по дате; экспорт.

### ADM (Administrator)
- **Только просмотр** — без создания пользователей, без журнала аудита, без смены ролей SUP; просмотр активности только VOL (уже так).
- **Пространства** — те же карточки и статистика по пространству (агрегат, не «только мои»), без импорта эмодзи и добавления пользователей.
- **Дашборд** — общая статистика по пространствам и сообщениям без привязки «только мои».
- **Список пользователей** — просмотр списка и карточек пользователей без опасных действий (блокировка, смена роли — только для VOL или ограниченно).

### VOL (Volunteer)
- **Один активный workspace** — уже ограничение; при блокировке — архивация его пространства, без возможности раархивировать; кнопка «Добавить пространство» неактивна при одном активном (уже так).
- **Срок доступа** — бейдж «Доступ до ДД.ММ.ГГГГ» в шапке и на карточках; предупреждение за N дней (уже есть в layout и на странице пространства).
- **«Ваш интенсив»** — бейдж на карточке и на странице пространства, если `volunteerIntensive` совпадает с URL/названием (уже есть).
- **Сообщения** — только свои; подпись «Показаны только ваши запланированные сообщения» (уже есть).
- **Скрытые элементы** — без «Проверить подключение», без редактирования пространства, без архивации, без истории (уже так).
- **Дашборд** — только свои пространства и свои сообщения; без фильтра «по пользователю».

## Приоритеты (условно)

1. **Высокий:** доработки по ролям (SUP/ADM/VOL), единые тултипы и подтверждения, адаптив сайдбара и таблиц.
2. **Средний:** хлебные крошки везде, «Забыли пароль», настраиваемые виджеты дашборда, экспорт в админке и на странице пространства.
3. **Низкий:** виртуализация списков, клавиатурные сокращения, расширенная доступность (aria), опционально 2FA и инвайты.

Можно внедрять поэтапно: сначала стабильность и роль-специфика, затем удобство (UI/UX), затем «по желанию».
